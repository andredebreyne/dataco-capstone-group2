# Report Folder Guide

This folder separates final-facing report materials from drafting and planning artifacts.

## Final-Facing Files

| Path | Purpose |
| --- | --- |
| `final_capstone_report_draft_v3.md` | Current final report draft for final formatting and table/figure insertion. |
| `final_capstone_report.md` | Earlier final-facing report draft retained because existing navigation links point to it. |
| `final_artifact_index.md` | Grader/reviewer navigation index for project evidence. |
| `final_validation_summary.md` | Final-facing validation status summary. |
| `final_report_assets/` | Final report tables, figure sources, dashboard screenshot placeholder, and manifests. |
| `tables/` | Analytical report tables generated by the project workflow and validators. |
| `figures/` | Analytical report figures generated by the project workflow. |

## Development and Drafting Files

Drafting files have been moved to `_drafting/` where safe. These include:

- old report drafts;
- expanded draft versions;
- outline and gap-check files;
- APA planning maps and inventories;
- APA TODO resolution plans;
- selected/verified APA reference planning files;
- draft change logs.

Two non-critical APA CSV planning files remain in this folder because they were locked during cleanup:

- `final_apa_todo_resolution_inventory.csv`
- `final_verified_apa_metadata.csv`

They can be moved to `_drafting/` later after the file lock is released.

## Proposed Organization

Use the root of `report/` for final-facing report navigation, final report drafts, validation summaries, and generated analytical tables/figures. Use `_drafting/` for development notes and planning artifacts that support review history but should not distract final graders. Use `final_report_assets/` for the report-ready table and figure package.

## Final Report Asset Package

The final report asset package is located at:

```text
report/final_report_assets/
```

It contains:

- seven non-dashboard final report tables in Markdown and CSV;
- a Mermaid source file for Figure 1;
- a placeholder instruction file for the final Power BI dashboard screenshot;
- source manifests and cleanup documentation.

No dashboard screenshot, `.pbix`, or fake Power BI artifact was created by the asset packaging task.
